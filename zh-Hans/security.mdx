# 安全

Slinq 从设计上就把安全放在比较前的位置。下面是平台主要的保护机制。

## 智能合约安全

### 锁定流动性

每一个 Slinq 代币都有 **永久锁定的流动性**：

- LP 会被发送到 LP Locker 合约
- 合约里没有 withdraw 功能
- 就连 Slinq 团队也无法解锁流动性
- 这能防止常见的 Rug Pull

<Check>
流动性一旦锁进去，就拿不出来。你的交易池是永久存在的。
</Check>

### 代币参数不可更改

代币发出之后，这些属性都不能再修改：

| 参数 | 可修改？ |
|------|-----------|
| 名称 | 否 |
| Symbol | 否 |
| 供应量 | 否 |
| 图片 | 否 |
| 创作者钱包 | 否 |

<Info>
不可变性可以防止项目方发币后再偷偷改名字、改形象、改参数去骗交易者。
</Info>

### 没有团队预留

Slinq 代币默认具备这些特点：
- 100% 供应量进入流动性池
- 0% 团队预留
- 0% 预挖
- 没有隐藏钱包

创作者赚的是交易手续费，不是靠预留筹码提前埋伏。

## MEV 保护

### 什么是 MEV？

MEV（Maximal Extractable Value）攻击通常包括：
- **抢跑（Front-running）：**机器人在你前面先买，吃掉价格冲击
- **夹子（Sandwiching）：**机器人先买后卖，把你夹在中间套利
- **狙击（Sniping）：**在新币开盘的前几个区块疯狂抢筹

### 我们怎么防

Slinq 的开盘保护会在前 30 秒对买单启用动态费率。总 hook 手续费上限是 5%，之后会回到 1% 的基础费率。

| 上线后的时间 | 买入费率 | 卖出费率 |
|--------------|----------|----------|
| 0 秒 | 5% | 1% |
| 10 秒 | ~3.2% | 1% |
| 20 秒 | ~1.6% | 1% |
| 30 秒后 | 1% | 1% |

开盘初期买入费率更高，能让很多 MEV 玩法直接变得不划算，给普通用户更公平的入场窗口。

## 手续费分发安全

### 智能合约托管

创作者收益会在链上记录，并且只能打到当前被设置为收益接收方的钱包：

- 不由 Slinq 团队人工控制
- 只有当前收益接收方钱包能收到领取结果
- 无法被重定向或盗走
- 全程链上可验证

### 提现安全

当你领取手续费时：
- 只能由你自己的钱包发起
- 资金直接打到你的钱包
- 中间没有托管人或中介

## 平台安全

### 钱包连接

Slinq 使用 **Privy** 来处理钱包连接：

- 我们不会看到或保存你的私钥
- 登录依赖加密签名
- Session token 会被加密
- 服务器上不存密码

### 基于签名的部署

发币流程基于 EIP-712 Typed Signature：

- 你签的是消息，不是链上交易
- 发币不需要 gas
- 有重放保护，避免重复部署
- 签名可以证明所有权，但不会暴露私钥

## 上报安全问题

如果你发现了漏洞：

1. **不要** 公开披露
2. 通过官方渠道联系团队
3. 提供可复现步骤
4. 给我们留出修复时间后再考虑披露

我们会认真对待所有负责任的安全报告。

## 风险提示

<Warning>
**meme 币属于高风险资产。** 即使有这些安全机制，也仍然存在下面这些风险：

- 代币价格可能直接归零
- 交易本身带有金融风险
- 过去表现不代表未来结果
- 只能投入你能完全亏掉的钱

Slinq 提供的是平台和机制，不替你做投资判断。
</Warning>
