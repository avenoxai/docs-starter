# 自定义你的代币

Web 界面可以让你把代币的视觉和文案都做得更像一个完整项目。下面是每个选项的最佳用法。

## 代币图片

你的代币图片会出现在列表、详情页和社交分享卡片里。别人第一眼看到的往往就是它。

### 图片要求

| 要求 | 规格 |
|------|------|
| 格式 | PNG 或 JPG |
| 最大体积 | 5 MB |
| 推荐尺寸 | 400x400 像素 |
| 宽高比 | 正方形（1:1） |

### 图片建议

<Columns cols={2}>
  <Card title="越简单越好" icon="circle">
    列表缩略图很小，简单、醒目的图形往往比复杂细节更有效。
  </Card>
  <Card title="对比度要高" icon="sun">
    图片最好在浅色和深色背景下都看得清。
  </Card>
  <Card title="少放字" icon="font">
    缩略图尺寸下，文字大概率会糊。图形通常比文案更有效。
  </Card>
  <Card title="风格统一" icon="palette">
    颜色和视觉语言统一，用户更容易记住你。
  </Card>
</Columns>

### AI 图片生成

如果你没有设计师，也可以直接用内置 AI：

1. 输入代币名称和描述
2. 点击 `Generate with AI`
3. 等几秒看结果
4. 不满意就重生成，或者直接上传你自己的图

<Info>
AI 生成的图片是为你的代币单独生成的，可以商用。
</Info>

## 代币名称

全名会出现在交易页和搜索结果里。

**一个好名字通常具备这些特点：**
- 好记，有辨识度
- 容易拼写，也容易读
- 和你的主题或社区有关
- 长度在 1-50 个字符之间

**示例：**
- `"Moon Token"` 对应 $MOON
- `"Degen Mode"` 对应 $DEGEN
- `"Based Protocol"` 对应 $BASED

## 代币 Symbol（Ticker）

Ticker 是交易者最常用来指代你代币的方式。

| 要求 | 说明 |
|------|------|
| 长度 | 1-10 个字符 |
| 字符 | 只能是字母和数字 |
| 大小写 | 系统会自动转成大写 |

**建议：**
- 越短越好，3-5 个字符最舒服
- 不要用特殊字符
- 发之前先确认没有被占

<Warning>
Ticker 是永久且唯一的。选之前想清楚，后面改不了。
</Warning>

## 描述

描述是交易者理解你这个币的第一段上下文。

### 可以写什么

- **目的：**这个币为什么存在？
- **社区：**它是给谁的？
- **主题：**它的 vibe 是什么？
- **用途：**如果有 utility，可以简单讲清楚

### 描述示例

<Tabs>
  <Tab title="Memecoin">
    ```
    $MOON 属于那批真拿得住的人。
    跌的时候不跑，涨的时候会喊。
    社区优先，LP 锁死，准备起飞。
    ```
  </Tab>
  <Tab title="社区代币">
    ```
    Based Builders 社区的官方代币。
    builder 做给 builder。
    一起在 BNB Chain 上持续把东西做出来。
    ```
  </Tab>
  <Tab title="创作者代币">
    ```
    用来支持我的内容创作路线。
    持有人可以更早看到视频、进专属频道，
    也能参与后续内容方向投票。
    ```
  </Tab>
</Tabs>

### AI 描述生成

也可以让 AI 先给你打一版草稿：

1. 输入代币名称
2. 点击 `Generate Description`
3. 再自己改一遍，把你的语气和梗补进去

## 社交链接

社交链接会大幅提升可信度，也方便别人找到你的社区。

| 链接类型 | 最适合谁 |
|----------|----------|
| **Website** | 已经有落地页或项目站的项目 |
| **Twitter/X** | 几乎所有代币都建议加，方便更新和互动 |
| **Telegram** | 做社群聊天和公告很方便 |

### 链接校验

Slinq 会在保存前验证这些链接：
- 必须是有效 URL（https://）
- 域名必须可访问
- 特殊字符会自动处理

<Tip>
就算你没有网站，至少也把 Twitter 补上。对交易者来说，这是最快的真实性校验方式。
</Tip>

## 初始 FDV

Fully Diluted Valuation 会决定你的初始市值和开盘价格。

### FDV 范围

| 设置项 | 范围 |
|--------|------|
| 最低 | 50 BNB |
| 最高 | 50,000 BNB |
| 默认 | 10 BNB |

### 怎么选 FDV

<Columns cols={3}>
  <Card title="低 FDV（50-200 BNB）" icon="seedling">
    **适合：**新创作者、小社区试验

    **优点：**门槛低，弹性大

    **缺点：**初始流动性会更浅
  </Card>
  <Card title="中 FDV（500-2,000 BNB）" icon="tree">
    **适合：**大多数项目

    **优点：**流动性和成长空间相对均衡

    **缺点：**没有特别极端的优势
  </Card>
  <Card title="高 FDV（5,000+ BNB）" icon="building">
    **适合：**已有影响力的创作者或大项目

    **优点：**流动性深，品牌感更强

    **缺点：**更依赖强社区来承接
  </Card>
</Columns>

### 价格计算

```
Token Price = Starting FDV ÷ 100,000,000,000（总供应量）
```

| FDV | 代币价格 |
|-----|----------|
| 50 BNB | 0.0000000005 BNB |
| 500 BNB | 0.000000005 BNB |
| 5,000 BNB | 0.00000005 BNB |
| 50,000 BNB | 0.0000005 BNB |

## 发币前检查清单

点击 `Launch` 之前，按这个清单过一遍：

<Steps>
  <Step title="再看一眼 Ticker">
    拼写对吗？够好记吗？确定是你要的那个写法吗？
  </Step>
  <Step title="检查图片">
    缩小后还好看吗？文件格式和尺寸都对吗？
  </Step>
  <Step title="重读描述">
    有没有错字？别人看完能不能秒懂你在讲什么？
  </Step>
  <Step title="点开每个社交链接">
    确认都能正常跳转。
  </Step>
  <Step title="确认 FDV">
    这真的是你想要的起始估值吗？
  </Step>
</Steps>

<Warning>
所有代币参数都是永久的。慢一点没关系，别着急发。
</Warning>
