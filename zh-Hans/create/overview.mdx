## 每个代币默认都会拿到什么

不管你用哪种方式发，每一个 Slinq 代币都会带这些核心机制：

| 特性 | 说明 |
|------|------|
| **固定供应量** | 1000 亿枚，不增发 |
| **锁定流动性** | LP 永久锁定，没法直接 Rug |
| **创作者分成** | 最高可拿全部交易手续费的 40% |
| **MEV 保护** | 开盘时通过智能 Hook 压制机器人 |
| **原生 DEX 交易** | 从第一笔交易开始就直接跑在 PancakeSwap V4 上 |
| **AI 驱动** | 社交发币场景下，名称、Ticker 和描述可自动生成 |

## Slinq 和别家不一样的地方

很多 launchpad 会先让代币跑 bonding curve，之后再“迁移”到 DEX。Slinq 不走这条路。Slinq 的代币 **从第一笔交易开始就原生跑在 PancakeSwap 上**。

这件事之所以做得到，是因为底层用了 PancakeSwap V4 的 **hooks**。它们会在每一笔交易发生时自动执行，用来处理手续费、MEV 保护和创作者收益分发。

## AI Agent 也可以发币

Slinq 提供了专门的 API，允许 AI Agent 和 Bot 以程序化方式发币。如果你在做 AI Agent、交易机器人或自动化工具，你可以：

- **通过 API 发币**，不用手动点页面
- **每天拿 1 次免 gas 赞助发币**，链上交易成本由协议承担
- **自定义收益接收方**，可以把创作者分成指定给任意钱包、Twitter 账号或 Farcaster 用户名
- **直接交易代币**，用预配置钱包通过 API 买卖

<Info>
AI Agent 发出的代币会和其他代币一起出现在 Slinq 上。它们拥有相同的基础机制：锁定流动性、创作者收益分成，以及通过 hooks 实现的 MEV 保护。
</Info>

## 频率限制

为了控制平台质量，每个用户每天有发币上限：

| 平台 | 每日上限 |
|------|----------|
| Twitter | 2 |
| Farcaster | 4 |
| Telegram | 3 |
| Web | 3 |

<Info>
限额会在 UTC 零点重置。如果你是正经项目需要更高额度，可以直接联系团队。
</Info>
